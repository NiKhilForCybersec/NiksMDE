Microsoft Defender for Endpoint Overview

Microsoft Defender for Endpoint (MDE) is Microsoft's enterprise-grade endpoint detection and response (EDR) platform. Originally known as Windows Defender ATP, MDE has evolved into a comprehensive endpoint security solution that provides preventive protection, post-breach detection, automated investigation, and response capabilities.

What is MDE?

MDE is a cloud-native endpoint security platform designed to help enterprise networks prevent, detect, investigate, and respond to advanced threats. It combines behavioral sensors, cloud security analytics, and threat intelligence to provide comprehensive endpoint protection.

EDR vs EPP vs XDR: Understanding the Terminology

Understanding these three concepts is crucial for any security professional:

Term Definition MDE Coverage
EPP
Endpoint Protection Platform		 Preventive security controls: antivirus, anti-malware, firewall, device control. Focus on blocking known threats. Microsoft Defender Antivirus, ASR rules, Exploit Protection, Network Protection
EDR
Endpoint Detection & Response		 Post-breach detection and investigation: behavioral monitoring, threat hunting, forensics, response actions. Behavioral sensors, Advanced Hunting, Live Response, Device Timeline
XDR
Extended Detection & Response		 Cross-domain detection correlating endpoints, email, identity, and cloud apps for unified threat visibility. Microsoft 365 Defender (MDE + MDO + MDI + MDA)

Key Insight

MDE provides both EPP and EDR capabilities in a single agent. When combined with other Microsoft Defender products (MDO, MDI, MDA), it becomes part of the XDR solution called Microsoft 365 Defender.

Licensing Note

  • Microsoft 365 A5 - Education equivalent of E5

Key Architecture Points

  • Cloud-Native: No on-premises infrastructure required; all processing happens in Microsoft cloud
  • Lightweight Sensor: Minimal endpoint footprint; most processing offloaded to cloud
  • Real-Time Telemetry: Continuous streaming of endpoint events to cloud for analysis
  • 30-Day Retention: Raw telemetry stored for 30 days (configurable up to 180 days)
  • Multi-Platform: Support for Windows, macOS, Linux, iOS, and Android
