KQL Queries for Reports

KQL - Reporting Queries 
// Monthly incident summary
AlertInfo
| where Timestamp > ago(30d)
| summarize 
    TotalAlerts = count(),
    Critical = countif(Severity == "Critical"),
    High = countif(Severity == "High"),
    Medium = countif(Severity == "Medium"),
    Low = countif(Severity == "Low")

// Week-over-week comparison
let ThisWeek = 
    AlertInfo
    | where Timestamp > ago(7d)
    | count;
let LastWeek = 
    AlertInfo
    | where Timestamp between (ago(14d) .. ago(7d))
    | count;
print ThisWeek=ThisWeek, LastWeek=LastWeek, 
      Change=ThisWeek - LastWeek

// Top affected business units (by device group)
AlertEvidence
| where Timestamp > ago(30d)
| where EntityType == "Machine"
| join kind=inner DeviceInfo on DeviceId
| summarize IncidentCount = dcount(AlertId) by MachineGroup
| top 10 by IncidentCount

// Detection source effectiveness
AlertInfo
| where Timestamp > ago(30d)
| summarize Count = count() by DetectionSource
| extend Percentage = round(100.0 * Count / toscalar(
    AlertInfo 
    | where Timestamp > ago(30d) 
    | count), 1)
| order by Count desc

Reporting Tips
  • Consistency: Use the same metrics and format each period
  • Trends matter: Direction is often more important than absolute numbers
  • Be honest: Don't hide bad news; present it with context and mitigation plan
  • Have backup detail: Be ready to drill down if asked
  • Automate: Schedule report generation to save time

ğŸ’¡ Key Interview Points

  • Audience adaptation: Executives want risk/business impact, SOC wants tactical data
  • Key executive metrics: Exposure score, breach probability, compliance status
  • Key operational metrics: MTTD, MTTR, alert backlog, false positive rate
  • Communication: Translate technical to business language
  • Reporting frequency: Board quarterly, CISO monthly, SOC daily/weekly
