MDE Glossary

This glossary covers key terms, acronyms, and concepts used in Microsoft Defender for Endpoint and related security technologies.

A

Terms A 
AIR (Automated Investigation and Response)
Automatically investigates alerts and can remediate threats without 
analyst intervention. Reduces manual workload and speeds response.

Alert
Notification of potentially malicious activity detected on an endpoint. 
Alerts have severity levels: Critical, High, Medium, Low, Informational.

AMSI (Antimalware Scan Interface)
Windows interface allowing applications to request malware scans of 
content. Used to detect fileless malware and malicious scripts.

ASR (Attack Surface Reduction)
Set of rules that block common attack techniques like Office macros 
spawning processes, executable email content, and script obfuscation.

ATP (Advanced Threat Protection)
Legacy name for Microsoft Defender for Endpoint. Still seen in some 
documentation and registry keys.

Advanced Hunting
Query-based threat hunting using KQL to search raw telemetry data. 
Allows proactive threat detection and investigation.

B-C

Terms B-C 
Behavioral Blocking
Detection based on process behavior rather than signatures. Identifies 
malicious actions regardless of file hash or reputation.

Block Mode (EDR)
EDR capability to block malicious artifacts even when third-party AV 
is primary. Acts as safety net for missed detections.

C2 (Command and Control)
Server infrastructure used by attackers to communicate with and control 
compromised endpoints. Key indicator of active compromise.

CFA (Controlled Folder Access)
Ransomware protection feature that blocks untrusted apps from modifying 
files in protected folders (Documents, Pictures, etc.).

Cloud Protection
Real-time cloud lookup for file reputation. Provides faster detection 
of new threats using Microsoft's threat intelligence.

Custom Detection
User-created detection rules using KQL queries. Runs on schedule and 
generates alerts when matches found.

D-E

Terms D-E 
Device Control
USB and removable media management. Can block, allow, or audit 
removable storage devices based on policy.

Device Isolation
Response action that cuts network connectivity except to MDE cloud. 
Contains threats while maintaining remote investigation capability.

EDR (Endpoint Detection and Response)
Technology that monitors endpoints for threats and provides 
investigation and response capabilities. Core MDE feature.

EPP (Endpoint Protection Platform)
Traditional antivirus and endpoint protection. MDE includes EPP 
(Microsoft Defender Antivirus) plus EDR capabilities.

Exploit Guard
Collection of features including ASR, Network Protection, CFA, and 
Exploit Protection. Successor to EMET.

Exploit Protection
System and application-level mitigations against memory corruption 
attacks. Includes DEP, ASLR, CFG protections.

Exposure Score
TVM metric (0-100) indicating organizational vulnerability. Lower 
scores indicate better security posture.

F-I

Terms F-I 
False Positive (FP)
Legitimate activity incorrectly flagged as malicious. Requires 
investigation to confirm and may need rule tuning.

Incident
Collection of related alerts representing a single attack. Groups 
alerts for coordinated investigation and response.

Indicator (IOC)
Indicator of Compromise - file hash, IP, domain, or URL known to be 
malicious. Can be used to block or alert on matches.

Investigation Package
Forensic data collection from endpoint. Includes autoruns, processes, 
network connections, event logs, and more.

Intune
Microsoft's cloud-based endpoint management. Primary method for 
deploying MDE policies to managed devices.

K-L

Terms K-L 
KQL (Kusto Query Language)
Query language used for Advanced Hunting. Similar to SQL but optimized 
for log analytics and time-series data.

Live Response
Remote shell access to endpoints for investigation and remediation. 
Allows running commands, collecting files, and taking action.

LOLBAS (Living Off the Land Binaries and Scripts)
Legitimate system tools used maliciously. Examples: PowerShell, 
certutil, mshta. Common technique to evade detection.

M-N

Terms M-N 
MAPS (Microsoft Active Protection Service)
Cloud-based protection service for real-time threat intelligence. 
Provides sample submission and reputation lookup.

MDE (Microsoft Defender for Endpoint)
Enterprise endpoint security platform. Includes EPP, EDR, TVM, and 
automated investigation capabilities.

MDI (Microsoft Defender for Identity)
Identity threat detection for Active Directory. Detects credential 
theft, lateral movement, and reconnaissance.

MDO (Microsoft Defender for Office 365)
Email and collaboration security. Protects against phishing, malware 
in attachments, and malicious links.

MDCA (Microsoft Defender for Cloud Apps)
Cloud access security broker (CASB). Monitors and controls cloud 
application usage and data.

Microsoft 365 Defender
Unified XDR platform combining MDE, MDI, MDO, and MDCA. Single 
portal for cross-domain detection and response.

MITRE ATT&CK
Framework of adversary tactics and techniques. Used to map and 
categorize threat behaviors.

MsSense.exe
MDE sensor service executable. Core process for EDR telemetry 
collection and reporting.

Network Protection
Blocks outbound connections to malicious sites. Extends SmartScreen 
protection to all applications.

O-R

Terms O-R 
Onboarding
Process of enrolling devices into MDE. Installs sensor and begins 
telemetry collection.

Offboarding
Process of removing devices from MDE. Stops sensor and clears 
local data.

P1/P2 (Plan 1/Plan 2)
MDE licensing tiers. P1 includes prevention only. P2 adds EDR, 
hunting, TVM, and automated investigation.

PUA (Potentially Unwanted Application)
Software that may be unwanted but isn't strictly malicious. 
Examples: adware, bundled software, crypto miners.

Quarantine
Isolation of malicious files. Prevents execution while preserving 
evidence for analysis.

Real-time Protection
Continuous scanning of files and processes as they execute. Core 
antivirus functionality.

S-T

Terms S-T 
Secure Score
Microsoft security posture metric. Measures implementation of 
security recommendations across Microsoft 365.

Sensor
MDE component on endpoints that collects telemetry. Runs as 
MsSense.exe service.

SmartScreen
URL and download reputation service. Blocks known malicious 
sites and files.

Suppression Rule
Rule to hide specific alerts from view. Used to reduce noise 
from known benign activity.

Tamper Protection
Prevents local changes to security settings. Blocks attempts 
to disable Defender via registry or PowerShell.

Threat Analytics
Threat intelligence reports on emerging threats. Includes 
exposure assessment and recommended mitigations.

Timeline
Chronological view of device activity. Shows processes, files, 
network, registry events for investigation.

True Positive (TP)
Legitimate detection of actual malicious activity. Confirms 
detection accuracy.

TVM (Threat and Vulnerability Management)
Vulnerability assessment and remediation tracking. Agentless 
discovery using MDE sensor data.

U-Z

Terms U-Z 
WdFilter.sys
Windows Defender file system filter driver. Intercepts file 
operations for real-time scanning.

Web Content Filtering
Category-based web blocking. Blocks access to sites by category 
(adult, gambling, etc.).

XDR (Extended Detection and Response)
Cross-domain detection and response. Microsoft 365 Defender 
provides XDR across endpoints, identity, email, and cloud apps.

Zero-Day
Previously unknown vulnerability with no available patch. 
Requires workarounds or mitigations until patched.

  • TVM: Vulnerability management without additional scanning
  • AIR: Automated investigation - virtual analyst capability
