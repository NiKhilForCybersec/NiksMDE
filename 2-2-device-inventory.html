Advanced Hunting - Devices

KQL - Device Queries 
// Get device inventory summary
DeviceInfo
| where Timestamp > ago(1d)
| summarize arg_max(Timestamp, *) by DeviceId
| summarize Count = count() by OSPlatform

// Find devices not seen recently
DeviceInfo
| summarize LastSeen = max(Timestamp) by DeviceName, DeviceId
| where LastSeen < ago(7d)
| order by LastSeen asc

// Devices with high risk level
DeviceInfo
| where Timestamp > ago(1d)
| summarize arg_max(Timestamp, *) by DeviceId
| where RiskScore == "High"
| project DeviceName, RiskScore, ExposureLevel, OSPlatform

// Devices by logged-on user
DeviceInfo
| where Timestamp > ago(1d)
| where isnotempty(LoggedOnUsers)
| extend Users = parse_json(LoggedOnUsers)
| mv-expand Users
| extend UserName = Users.UserName
| summarize Devices = make_set(DeviceName) by tostring(UserName)

Best Practices
  • Monitor health daily: Check for inactive or misconfigured devices
  • Use tags: Organize devices by department, criticality, or function
  • Create device groups: Enable targeted policies and RBAC
  • Review high-risk devices: Prioritize investigation and patching
  • Export for reporting: Use export feature for compliance reports

ğŸ’¡ Key Interview Points

  • Health states: Active, Inactive, Misconfigured, Impaired, No sensor data
  • Risk vs Exposure: Risk = active threats, Exposure = vulnerabilities
  • Device page tabs: Overview, Alerts, Timeline, Software, Vulnerabilities
  • Device groups: Rule-based, used for RBAC and policy targeting
  • Response actions: Isolate, scan, restrict apps, collect package
