Custom Reporting with KQL

KQL - Custom Report Queries 
// Weekly alert summary
AlertInfo
| where Timestamp > ago(7d)
| summarize 
    Total = count(),
    Critical = countif(Severity == "Critical"),
    High = countif(Severity == "High"),
    Medium = countif(Severity == "Medium"),
    Low = countif(Severity == "Low")

// Daily alert trend for charting
AlertInfo
| where Timestamp > ago(30d)
| summarize AlertCount = count() by bin(Timestamp, 1d), Severity
| render timechart

// Top 10 alerting devices
AlertEvidence
| where Timestamp > ago(30d)
| where EntityType == "Machine"
| summarize AlertCount = count() by DeviceName
| top 10 by AlertCount

// Detection source effectiveness
AlertInfo
| where Timestamp > ago(30d)
| summarize Count = count() by DetectionSource
| render piechart

// MITRE ATT&CK technique distribution
AlertInfo
| where Timestamp > ago(30d)
| mv-expand AttackTechniques
| summarize Count = count() by tostring(AttackTechniques)
| top 15 by Count

Reporting Best Practices
  • Know your audience: Executives want trends and risks, not raw data
  • Show comparisons: This month vs last, before/after changes
  • Highlight action items: What needs attention now
  • Automate where possible: Scheduled reports save time
  • Visualize trends: Charts are easier to understand than tables

ğŸ’¡ Key Interview Points

  • Built-in reports: Threat protection, device health, vulnerability, web protection
  • Custom reports: Use KQL in Advanced Hunting, export to CSV/Power BI
  • Executive reporting: Focus on trends, risks, and action items
  • Power BI: Connect via Log Analytics or API for advanced dashboards
  • Key metrics: Exposure score, MTTD, MTTR, alert trends
