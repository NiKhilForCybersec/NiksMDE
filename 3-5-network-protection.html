Network Protection

Network Protection extends Microsoft Defender SmartScreen protection to block outbound HTTP(S) traffic to low-reputation destinations, protecting against phishing, exploits, and malicious content regardless of which application initiates the connection.

Configuration Methods

PowerShell Configuration

PowerShell - Network Protection 
# Check current status
Get-MpPreference | Select-Object EnableNetworkProtection

# Enable Network Protection (Block mode)
Set-MpPreference -EnableNetworkProtection Enabled

# Enable in Audit mode (recommended for testing)
Set-MpPreference -EnableNetworkProtection AuditMode

# Enable in Warn mode
Set-MpPreference -EnableNetworkProtection Warn

# Disable Network Protection
Set-MpPreference -EnableNetworkProtection Disabled

# Verify configuration
(Get-MpPreference).EnableNetworkProtection
# 0 = Disabled, 1 = Enabled, 2 = Audit

# Check if SmartScreen is enabled (required)
Get-ItemProperty "HKLM:\SOFTWARE\Microsoft\Windows\CurrentVersion\Explorer" -Name SmartScreenEnabled

Common Issues
  • Legitimate site blocked: Check if miscategorized, submit for review, or add allow indicator
  • No blocks appearing: Verify Network Protection is enabled and SmartScreen is on
  • Proxy issues: May need to configure proxy settings for SmartScreen cloud lookups
  • Browser-only protection: Some browsers use their own filtering - Network Protection works for all apps

ğŸ’¡ Key Interview Points

  • Purpose: Block outbound connections to malicious sites from ANY application
  • Difference from SmartScreen: SmartScreen is browser-only, Network Protection is system-wide
  • Modes: Disabled, Enabled (block), Audit, Warn
  • Sources: SmartScreen reputation, custom indicators, web content filtering
  • Prerequisite: SmartScreen and cloud protection must be enabled
  • Works for: HTTP/HTTPS traffic from any application
