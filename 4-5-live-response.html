Live Response

Live Response provides security teams with instant remote shell access to endpoints directly from the MDE portal. This enables real-time investigation, evidence collection, and remediation without requiring physical access or traditional remote desktop tools.

Permission Requirements

Capability Required Role Commands Allowed
Basic Live Response Security Reader + Live Response Read-only commands (dir, connections, processes)
Advanced Live Response Security Operator or Admin All commands including run, putfile, remediate

Starting a Live Response Session

  1. Navigate to Device Inventory or Device Page
  2. Click the device name to open device details
  3. Click "Initiate Live Response Session" button
  4. Wait for connection (may take 30-60 seconds)
  5. Session opens in new console window

Session Limits

Maximum 25 concurrent Live Response sessions per tenant. Sessions auto-timeout after 30 minutes of inactivity.

Basic Commands

These commands are available with basic Live Response permissions:

Command Description Example
help Show available commands help
cd Change directory cd C:\Users\Public
dir List directory contents dir C:\Windows\Temp
cls Clear screen cls
connections Show active network connections connections
processes List running processes processes
scheduledtasks List scheduled tasks scheduledtasks
services List Windows services services
drivers List loaded drivers drivers
registry Query registry values registry HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Run
trace Set terminal logging mode trace enable

Advanced Commands

These commands require Advanced Live Response permissions:

Command Description Example
getfile Download file from endpoint getfile C:\Windows\Temp\malware.exe
putfile Upload file to endpoint putfile remediation.ps1
run Execute script from library run CollectLogs.ps1
remediate Remediate an entity remediate file C:\malware.exe
undo Restore remediated entity undo file C:\malware.exe
analyze Submit file for deep analysis analyze C:\suspicious.exe
findfile Search for files by name findfile *.ps1

Workflow 1: Initial Triage

Workflow 2: File Collection for Analysis

Live Response

dir

# Step 4: Submit for deep analysis

# Step 5: Collect additional artifacts

Workflow 3: Remediation

Live Response 
# Step 1: Identify malicious file
dir C:\Users\Public\Downloads

# Step 2: Remediate (quarantine) the file
remediate file C:\Users\Public\Downloads\malware.exe

# Step 3: Remove persistence
run RemoveScheduledTask.ps1 -parameters "-TaskName MaliciousTask"

# Step 4: If needed, undo remediation
undo file C:\Users\Public\Downloads\malware.exe

Sample Scripts for Library

PowerShell - CollectArtifacts.ps1 
# CollectArtifacts.ps1 - Collect common forensic artifacts
param(
    [string]$OutputPath = "C:\Windows\Temp\Artifacts"
)

# Create output directory
New-Item -Path $OutputPath -ItemType Directory -Force

# Collect running processes
Get-Process | Select-Object Name, Id, Path, StartTime | 
    Export-Csv "$OutputPath\processes.csv" -NoTypeInformation

# Collect network connections
Get-NetTCPConnection | 
    Export-Csv "$OutputPath\connections.csv" -NoTypeInformation

# Collect scheduled tasks
Get-ScheduledTask | 
    Export-Csv "$OutputPath\scheduledtasks.csv" -NoTypeInformation

# Collect services
Get-Service | Select-Object Name, Status, StartType, DisplayName |
    Export-Csv "$OutputPath\services.csv" -NoTypeInformation

# Collect autorun locations
Get-ItemProperty "HKLM:\SOFTWARE\Microsoft\Windows\CurrentVersion\Run" |
    Out-File "$OutputPath\autoruns_hklm.txt"

Write-Output "Artifacts collected to $OutputPath"

Live Response Best Practices
  • Always document commands run during investigation
  • Use trace enable to log session activity
  • Collect evidence before remediation
  • Test scripts in library before deploying to production
  • Coordinate with endpoint user if system is in active use

When to Use Live Response

  • Investigating active alerts requiring immediate endpoint access
  • Collecting files for malware analysis
  • Verifying remediation actions completed successfully
  • Gathering additional evidence during incident response
  • Running pre-approved remediation scripts

Common Tickets You'll Handle

ğŸ’¡ Key Interview Points

  • Two permission levels: Basic (read-only) and Advanced (remediation, script execution)
  • Key commands: processes, connections, getfile, run, remediate
  • Session limits: 25 concurrent sessions, 30-minute timeout
  • Script library: Upload reusable scripts for consistent remediation
  • Evidence first: Always collect files before remediation
  • Documentation: Use trace enable to log all commands for case documentation
