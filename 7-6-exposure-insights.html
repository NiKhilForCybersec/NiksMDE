Advanced Hunting for Exposure

KQL Queries
// Devices with critical vulnerabilities
DeviceTvmSoftwareVulnerabilities
| where VulnerabilitySeverityLevel == "Critical"
| summarize CriticalCVEs = dcount(CveId) by DeviceName, DeviceId
| order by CriticalCVEs desc

// Most common critical CVEs
DeviceTvmSoftwareVulnerabilities
| where VulnerabilitySeverityLevel == "Critical"
| summarize AffectedDevices = dcount(DeviceId) by CveId, SoftwareName
| order by AffectedDevices desc
| take 20

// Devices with end-of-support software
DeviceTvmSoftwareInventory
| where EndOfSupportStatus == "EOS Version" or EndOfSupportStatus == "EOS Software"
| summarize by DeviceName, SoftwareName, SoftwareVersion, EndOfSupportDate

// Security configuration compliance
DeviceTvmSecureConfigurationAssessment
| where IsCompliant == false
| summarize NonCompliantCount = count() by ConfigurationId, ConfigurationName
| order by NonCompliantCount desc

Best Practices
  • Weekly review: Check exposure score and recommendations weekly
  • Critical first: Prioritize critical CVEs, especially with exploits
  • Track as KPI: Report exposure score to leadership
  • Automate patching: Use automated deployment where possible
  • Set targets: Define exposure score goals (e.g., below 30)
