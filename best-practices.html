ASR Rules - Recommended Configuration

Block Mode (Minimal Impact):

  • Block executable content from email
  • Block Office apps from creating executable content
  • Block Office apps from injecting code
  • Block obfuscated scripts
  • Block JS/VBS from launching executables
  • Block credential stealing from LSASS
  • Block ransomware advanced protection
  • Block Adobe Reader child processes

Audit First (Test Before Blocking):

  • Block Office apps from creating child processes
  • Block Win32 API calls from Office macros
  • Block PSExec and WMI process creation
  • Block low-prevalence executables

PowerShell Hardening Script

PowerShell
# MDE Hardening Script - Run as Administrator
# Enable maximum protection settings

# Cloud Protection
Set-MpPreference -MAPSReporting Advanced
Set-MpPreference -SubmitSamplesConsent SendAllSamples
Set-MpPreference -DisableBlockAtFirstSeen $false
Set-MpPreference -CloudBlockLevel High
Set-MpPreference -CloudExtendedTimeout 50

# Real-Time Protection
Set-MpPreference -DisableRealtimeMonitoring $false
Set-MpPreference -DisableBehaviorMonitoring $false
Set-MpPreference -DisableIOAVProtection $false

# PUA Protection
Set-MpPreference -PUAProtection Enabled

# Network Protection
Set-MpPreference -EnableNetworkProtection Enabled

# Verify settings
Get-MpPreference | Select-Object MAPSReporting, SubmitSamplesConsent, 
    CloudBlockLevel, PUAProtection, EnableNetworkProtection

Exclusion Guidelines
